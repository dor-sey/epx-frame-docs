# 低代码平台配置说明

## 1. 平台概述

本低代码平台采用声明式JSON配置驱动页面渲染，通过标准化的配置协议，实现快速构建企业级应用页面。平台核心设计理念是**配置即代码**，通过结构化的JSON描述页面布局、组件、数据绑定和交互逻辑，引擎动态解析并渲染为完整用户界面。

## 2. 配置核心结构

### 2.1 顶层配置对象
每个页面配置文件是一个JSON对象，包含以下核心字段：

| 字段名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| **`version`** | 字符串 | 是 | 配置协议版本，当前为"2.0" |
| **`js`** | 字符串 | 否 | 页面级自定义JS代码 |
| **`compileJS`** | 字符串 | 否 | 页面级自定义JS编译代码 |
| **`css`** | 字符串 | 否 | 页面级CSS样式，支持`:current`伪类 |
| **`layout`** | 字符串 | 是 | 页面布局模式：`flex`(弹性盒子)、`grid`(网格)、`absolute`(绝对定位)等 |
| **`items`** | 数组 | 是 | 页面组件定义数组，按渲染顺序排列 |
| **`itemConfigs`** | 对象 | 否 | 组件详细配置映射表，键名为组件`id` |

### 2.2 组件基础定义 (`items`数组)
每个组件定义包含以下通用属性：

```json
{
  "id": "component_unique_id",          // 组件唯一标识符（必填）
  "contentType": "component_type",      // 组件类型标识（必填）
  "title": "组件显示标题",              // 组件标题，用于显示和文档
  "componentName": "组件名称",          // 组件内部名称
  "css": "组件私有样式",                // 组件级CSS，覆盖页面样式
  "flex": true/false                    // 是否参与flex布局
}
```

### 2.3 组件详细配置 (`itemConfigs`对象)
为特定组件提供深度配置，结构因`contentType`而异：

```json
{
  "component_id": {
    // 数据相关配置
    "metaId": "元数据id",
    "filterFields": ["字段1", "字段2"],  
    "orderFields": ["排序字段"],
    // 迭代器配置（用于列表渲染）
    "iterator": {
      "name": "item",
      "index": "index"
    },
    ...
  }
}
```

## 3. 核心配置详解

### 3.1 数据绑定与模型
- **`metaId`**：关联后台数据模型的唯一标识，平台通过此ID获取数据结构和权限
- **数据过滤**：支持多条件组合过滤，`categoryScopeParams`定义数据查询条件
- **动态数据**：支持表达式绑定，如`{{currentUser.department}}`

### 3.2 布局与样式系统
- **弹性布局**：通过`layout: "flex"`和组件`flex`属性实现响应式布局
- **样式隔离**：页面级CSS和组件级CSS分离，支持样式继承和覆盖
- **主题支持**：通过CSS变量支持主题切换

## 4. 配置实例解析

以下是一个完整的人员信息管理页面配置：

```json
{
  "version": "2.0",
  "css": ":current {}",
  "layout": "flex",
  "items": [
    {
      "title": "导航",
      "id": "nav",
      "contentType": "header"
    },
    {
      "id": "entity-list_myizliy5mu",
      "contentType": "entity-list",
      "title": "人员基本信息组织机构",
      "componentName": "人员基本信息组织机构",
      "css": ":current {\n  padding: default;\n}\n",
      "flex": true
    }
  ],
  "itemConfigs": {
    "entity-list_myizliy5mu": {
      "orderFields": [],
      "filterFields": [],
      "showCategoryNav": true,
      "categoryField": ["MAIN_DEPT"],
      "categoryOrderFields": ["UP", "ORDER_NUM"],
      "categoryExpandLevel": "first",
      "metaId": "3725ED0009C34D5480E9D1F5D685B129",
      "treeExpandLevel": "first",
      "personListPageId": "4CC036BAC6D843DD91AE607FA3E9EDC1",
      "iterator": {
        "name": "item",
        "index": "index"
      },
      "treeExpandMore": false,
      "categoryScopeParams": [
        {
          "lb": "",
          "field": "REMOVE_FLAG",
          "mode": "not_equal",
          "values": "1",
          "rb": "",
          "text": "是否撤销   不等于   是",
          "linkLabel": "",
          "values#": "是"
        }
      ],
      "categoryNavWidth": 260
    }
  }
}
```

### 4.1 实例功能解析

1. **页面结构**：
   - 顶部导航栏(`header`类型组件)
   - 主内容区为人员列表(`hr-ryxx-person-list`自定义组件)

2. **数据流程**：
   - 绑定数据模型`3725ED0009C34D5480E9D1F5D685B129`
   - 仅显示`REMOVE_FLAG ≠ 1`(未撤销)的人员记录
   - 按`MAIN_DEPT`(主部门)字段分类展示

3. **用户交互**：
   - 左侧显示部门分类导航树，宽度260px
   - 点击人员项跳转到详情页`4CC036BAC6D843DD91AE607FA3E9EDC1`
   - 列表项通过`iterator`定义渲染变量

4. **业务逻辑**：
   - 分类按`UP`(上级)和`ORDER_NUM`(排序号)排序
   - 默认展开一级部门(`categoryExpandLevel: "first"`)
   - 不显示"展开更多"按钮(`treeExpandMore: false`)
