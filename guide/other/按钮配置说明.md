# 按钮配置说明

## 1. 模型概述

本低代码平台的按钮配置模型采用**声明式JSON配置**实现复杂按钮行为的可视化配置。该模型支持**200+配置项**，涵盖按钮外观、交互、业务逻辑、权限控制等全场景。

## 2. 核心配置结构

每个按钮配置项遵循统一结构：

```javascript
{
    "text": "按钮名称",
    "ui": "按钮样式",
    "handler": "按钮事件",
    // 其他类型特定字段...
}
```

## 3. 配置项分类详解

### 3.1 基础外观配置
控制按钮的视觉表现和基本行为：

| 配置项 | 类型 | 描述 | 示例值 |
|--------|------|------|--------|
| **text** | text | 按钮显示文本 | `"保存"` |
| **ui** | ui-select | 按钮类型/样式 | `primary`/`success`/`info`等7种 |
| **styles** | checkbox | 样式组合 | `["plain", "round"]` |
| **icon** | object | 图标选择 | 从`FM_FONT_ICON`库选择 |
| **showName** | switch | 是否显示文本 | `true`/`false` |
| **terminal** | radio | 适用终端 | `"all"`/`"pc"`/`"mobile"` |

### 3.2 事件与动作配置
定义按钮点击后的核心行为：

| 配置项 | 类型 | 描述 | 依赖条件 |
|--------|------|------|----------|
| **handler** | select | 主事件处理器 | 基础必填 |
| **beforeTrigger** | function | 前置处理函数 | 所有handler |
| **afterTrigger** | select | 后置动作 | 多个handler |
| **function** | function | 自定义JS函数 | `handler: 'customJs'` |

### 3.3 页面导航配置
控制页面跳转行为：

| 配置项 | 类型 | 描述 | 触发条件 |
|--------|------|------|----------|
| **page** | page | 目标页面 | `handler: ['goPage', 'addEntity', ...]` |
| **menuId** | object | 目标菜单 | `handler: ['goPage', 'addEntity', 'editEntity']` |
| **openMode** | radio | 打开方式 | `handler: ['goPage', 'addEntity', ...]` |
| **pageType** | radio | 页面状态 | `handler: ['goPage', 'addEntity', 'editEntity']` |
| **backPageDelta** | num | 返回页面数 | `afterTrigger: ['backPage', 'backRefeshPage']` |

### 3.4 数据操作配置
与后台数据交互相关：

| 配置项 | 类型 | 描述 | 说明 |
|--------|------|------|------|
| **metaId** | entity | 数据实体 | 指定操作的数据对象 |
| **action** | action | 数据操作 | 控制权限和可见性 |
| **rule** | rule | 业务规则 | 执行业务验证逻辑 |

### 3.5 打印与报表配置
实现打印和报表导出功能：

| 配置项 | 类型 | 描述 | 逻辑关系 |
|--------|------|------|----------|
| **printWay** | radio | 打印方式 | 分支点：`custom` vs `fine` |
| **printCode** | object | 自定义模板 | `printWay: 'custom'` |
| **cptName** | object | 帆软模板 | `printWay: 'fine'` |
| **printOperates** | checkbox | 输出选项 | 下载PDF/Word/Excel/打印 |
| **downloadReportFormat** | radio | 文件格式 | 报表下载格式 |

### 3.6 流程审批配置
工作流相关功能：

| 配置项 | 类型 | 描述 | 功能说明 |
|--------|------|------|----------|
| **destActivityId** | process | 目标审批节点 | 指定审批流程节点 |
| **processNode** | process | 流程节点 | 申请流程节点 |
| **auditorOptional** | radio | 审核人自选 | 是否允许选择审核人 |
| **openOpinion** | radio | 意见弹窗 | 是否显示审核意见对话框 |
| **showCheckFields** | checkbox | 显示字段 | 审核操作/意见/签名 |

### 3.7 条件控制配置
实现动态显示和验证：

| 配置项 | 类型 | 描述 | 应用场景 |
|--------|------|------|----------|
| **limitFlag** | radio | 操作限制 | 控制按钮是否可用 |
| **limitCondition** | scope | 限制条件 | 复杂条件表达式 |
| **showConditionType** | radio | 条件类型 | `scope`(高级筛选)或`el`(表达式) |
| **showCondition** | scope | 显示条件 | 控制按钮可见性 |
| **confirm** | radio | 二次确认 | 操作前弹出确认对话框 |

### 3.8 反馈与消息配置
用户交互反馈：

| 配置项 | 类型 | 描述 | 默认值 |
|--------|------|------|--------|
| **successMsg** | text | 成功提示 | `"处理成功"` |
| **failMsg** | text | 失败提示 | `"处理失败"` |
| **confirmMsg** | text | 确认消息 | 自定义 |
| **mask** | radio | 加载遮罩 | 请求时显示loading |

## 4. 使用流程示例

### 4.1 配置一个"提交审核"按钮
1. **基础设置**：
   - `text`: "提交审核"
   - `ui`: "primary" (主要按钮)
   - `icon`: 选择审核图标

2. **事件配置**：
   - `handler`: 选择"applyEntity" (申请实体)
   - `metaId`: 选择数据对象
   - `processNode`: 选择审批流程节点

3. **条件控制**：
   - `limitFlag`: true (启用操作限制)
   - `limitCondition`: 设置提交条件
   - `confirm`: true (需要二次确认)
   - `confirmMsg`: "确定提交审核吗？"

4. **后置动作**：
   - `afterTrigger`: "backPage" (返回上一页)
   - `backPageDelta`: 1
   - `successMsg`: "提交成功"

### 4.2 配置一个"导出报表"按钮
1. **基础设置**：
   - `text`: "导出报表"
   - `ui`: "success" (成功按钮样式)

2. **事件配置**：
   - `handler`: "downloadReport"
   - `printWay`: "custom" (自定义报表)

3. **报表配置**：
   - `printCode`: 选择报表模板
   - `downloadReportFormat`: "excel"
   - `downloadReportWay`: "multiple" (多文件)

4. **文件命名**：
   - `downloadReportNameFields`: 选择命名字段
   - `downloadReportNameSplit`: "-"
