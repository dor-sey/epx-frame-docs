# 组件规范

## 命名规范

组件名称使用 **kebab-case**（短横线命名法）进行命名。自定义组件统一使用 **`epx-`** 作为前缀，以区分第三方组件和自定义组件。

### 命名格式

```
epx-<组件名称>
```

### 命名示例

| 组件类型 | 命名示例 | 说明 |
|----------|----------|------|
| 自定义组件 | `epx-add-form` | 新增编辑表单组件 |
| 自定义组件 | `epx-entity-list` | 对象列表组件 |
| 自定义组件 | `epx-text` | 文本展示组件 |
| 第三方组件 | `el-button` | Element UI 按钮 |
| 第三方组件 | `van-cell` | Vant 单元格 |

### 命名规则说明

1. **前缀**：所有自定义组件必须以 `epx-` 开头
2. **分隔符**：使用短横线 `-` 分隔单词
3. **语义化**：组件名称应能清晰表达组件的功能和用途
4. **唯一性**：组件名称在项目中必须唯一，避免命名冲突

::: warning 注意
不遵循命名规范的组件可能导致注册失败或组件无法正常渲染。请务必按照规范命名组件。
:::

## 文件组织

组件文件应按照功能模块进行组织，建议的目录结构如下：

```
src/
├── components/
│   ├── add-form/
│   │   ├── index.vue
│   │   └── add-form-item.vue
│   ├── entity-list/
│   │   ├── index.vue
│   │   └── entity-sub-list.vue
│   └── ...
```

## 组件注册

自定义组件需要在 `epx-frame` 中进行注册后才能使用。注册时组件名称必须与文件中定义的 `name` 属性一致：

```js
// 组件定义
export default {
  name: 'epx-my-component',
  // ...
}

// 注册组件
import MyComponent from './my-component.vue'
// 注册到 epx-frame 组件库中
```
