# 页面

页面是由**页面模板**和**页面配置内容**组成的。通过不同的页面配置，页面模板可以渲染出不同的页面效果。这种设计模式将页面的结构（模板）与内容（配置）分离，使得同一套模板可以服务于多种业务场景，极大提高了代码复用率。

## 页面组成

```
页面 = 页面模板 + 页面配置
```

- **页面模板**：定义页面的整体结构和布局，决定页面由哪些组件组成以及组件的排列方式。
- **页面配置**：定义页面的具体内容，包括组件的属性、数据源、交互行为等。

## 常用页面模板

EPX-Frame 内置了多种常用页面模板，覆盖了常见的业务场景：

| 模板名称 | 说明 |
|----------|------|
| `entity-add-page` | 对象新增页面 |
| `entity-list-page` | 对象列表页面 |
| `entity-tree-manage` | 对象管理页面 |
| `entity-list-find-page` | 对象查找页面 |

### 对象新增页面

用于创建新的业务数据记录。页面通常由**标题导航组件**和**新增编辑表单组件**组成，提供表单填写、数据校验和提交功能。

### 对象列表页面

用于展示和管理业务数据列表。页面通常由**标题导航组件**和**对象列表组件**组成，支持数据查询、筛选、分页、新增、编辑、删除、导入导出等操作。

### 对象管理页面

用于以树形结构管理业务数据，适用于有层级关系的数据管理场景，如组织架构、分类管理等。

### 对象查找页面

用于在列表中选择一条或多条数据记录，常用于关联选择、数据引用等场景。该页面继承自对象列表组件，并重写了部分变量和方法。
